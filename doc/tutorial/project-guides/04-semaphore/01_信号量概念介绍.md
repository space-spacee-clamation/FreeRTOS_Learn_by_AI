# 队列概念介绍

## 什么是队列？

### 生活中的类比

想象银行的排队系统：

- **顾客**：要处理的数据
- **排队队列**：顾客按顺序排队
- **窗口**：从队列中叫号处理
- **先来先服务**：先到的顾客先处理

在FreeRTOS中：
- **数据** = 顾客
- **队列** = 排队队伍
- **发送任务** = 顾客来排队
- **接收任务** = 窗口叫号处理

### 队列的定义

**队列（Queue）** 是FreeRTOS提供的一种任务间通信机制：

1. **数据缓冲区**：用来存储要传递的数据
2. **FIFO**：先进先出（First In, First Out）
3. **线程安全**：自动处理多任务访问的同步问题
4. **阻塞机制**：任务可以等待队列有数据或有空间

## 为什么需要队列？

### 没有队列时的问题

在没有队列的情况下，任务间传递数据可能会这样：

```c
// 全局变量（不推荐！）
uint32_t g_sharedData = 0;

void vSenderTask(void *p)
{
    for(;;)
    {
        g_sharedData = 某个值;  // 直接写全局变量
        vTaskDelay(100);
    }
}

void vReceiverTask(void *p)
{
    for(;;)
    {
        处理(g_sharedData);  // 直接读全局变量
        vTaskDelay(100);
    }
}
```

**问题**：
1. **数据竞争**：两个任务同时访问可能出错
2. **数据丢失**：Sender发太快，Receiver可能漏掉
3. **无法同步**：不知道什么时候有新数据
4. **难以调试**：出问题很难查

### 使用队列的优势

```c
QueueHandle_t xQueue = NULL;

void vSenderTask(void *p)
{
    for(;;)
    {
        uint32_t data = 某个值;
        xQueueSend(xQueue, &data, portMAX_DELAY);  // 发送到队列
        vTaskDelay(100);
    }
}

void vReceiverTask(void *p)
{
    for(;;)
    {
        uint32_t data;
        xQueueReceive(xQueue, &data, portMAX_DELAY);  // 从队列接收
        处理(data);
    }
}
```

**优势**：
1. **线程安全**：FreeRTOS自动处理同步
2. **数据缓冲**：Sender可以比Receiver快，数据不会丢
3. **任务同步**：Receiver可以等待数据到来
4. **易于理解**：逻辑清晰，易于维护

## 队列的工作原理

### FIFO（先进先出）

```
队列有5个位置：[ ][ ][ ][ ][ ]

发送数据1 → [1][ ][ ][ ][ ]  ← 数据1在最前面
发送数据2 → [1][2][ ][ ][ ]
发送数据3 → [1][2][3][ ][ ]

接收数据 ← [2][3][ ][ ][ ]  ← 数据1被取走（先到先出）
接收数据 ← [3][ ][ ][ ][ ]  ← 数据2被取走
```

### 队列的状态

#### 1. 空队列
```
[ ][ ][ ][ ][ ]
 ↑
队列为空，Receiver会阻塞等待
```

#### 2. 有数据的队列
```
[1][2][3][ ][ ]
 ↑  ↑
 头  尾
Receiver可以读取数据1
```

#### 3. 满队列
```
[1][2][3][4][5]
           ↑
        队列已满
Sender会阻塞等待空间
```

## 队列的阻塞机制

### 发送时的阻塞

当队列满时，发送任务可以选择：

1. **不等待**：立即返回错误
2. **等待一段时间**：等指定的时钟节拍数
3. **无限等待**：一直等直到有空间

```c
// 不等待
xQueueSend(xQueue, &data, 0);

// 等待100ms
xQueueSend(xQueue, &data, pdMS_TO_TICKS(100));

// 无限等待
xQueueSend(xQueue, &data, portMAX_DELAY);
```

### 接收时的阻塞

当队列为空时，接收任务可以选择：

1. **不等待**：立即返回错误
2. **等待一段时间**：等指定的时钟节拍数
3. **无限等待**：一直等直到有数据

```c
// 不等待
xQueueReceive(xQueue, &data, 0);

// 等待100ms
xQueueReceive(xQueue, &data, pdMS_TO_TICKS(100));

// 无限等待
xQueueReceive(xQueue, &data, portMAX_DELAY);
```

## 队列的数据存储

### 拷贝存储，不是引用

**重要**：队列存储的是数据的拷贝，不是指针！

```c
void vSenderTask(void *p)
{
    for(;;)
    {
        uint32_t ulLocalData = 某个值;  // 局部变量
        
        // 队列会拷贝ulLocalData的值，不是保存它的地址
        xQueueSend(xQueue, &ulLocalData, portMAX_DELAY);
        
        // 即使ulLocalData在这里被销毁，队列里的数据还在
        vTaskDelay(100);
    }
}
```

### 可以存储任意类型的数据

队列可以存储：

- **基本类型**：int, float, char等
- **结构体**：复杂的数据结构
- **指针**：指向数据的指针（要小心！）

```c
// 存储整数
xQueueCreate(10, sizeof(uint32_t));

// 存储结构体
typedef struct {
    uint32_t id;
    float value;
    char name[20];
} SensorData_t;

xQueueCreate(5, sizeof(SensorData_t));
```

## 队列的典型应用场景

### 1. 数据传递

```
传感器任务 → [队列] → 处理任务 → [队列] → 显示任务
```

### 2. 任务同步

```
触发事件 → 发送到队列 → 等待队列 → 处理事件
```

### 3. 工作队列

```
多个任务发送工作请求 → [队列] → 一个工作任务处理
```

### 4. 中断与任务通信

```
中断 → [队列] → 任务（中断中不能做复杂处理）
```

## 重要配置选项

在 FreeRTOSConfig.h 中：

```c
// 启用队列（默认是启用的）
#define configUSE_QUEUE_SETS          0

// 启用队列注册表（用于调试）
#define configQUEUE_REGISTRY_SIZE     10
```

## 总结

- **队列**：任务间传递数据的安全方式
- **FIFO**：先进先出的数据结构
- **线程安全**：自动处理同步问题
- **阻塞机制**：可以等待队列有数据或有空间
- **数据拷贝**：队列存储数据的副本，不是引用
