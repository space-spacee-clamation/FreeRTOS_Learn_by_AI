# 堆内存管理概念介绍

## 什么是堆内存管理？

### 生活中的类比

想象一个仓库管理系统：

- **仓库** = 系统的RAM内存
- **箱子** = 需要存储的数据
- **管理员** = 内存分配器
- **租/退箱子** = 分配/释放内存

在FreeRTOS中：
- **堆（Heap）** = 预留出来的一块内存区域
- **内存分配器** = 管理这块内存的算法
- **pvPortMalloc()** = 申请内存
- **vPortFree()** = 释放内存

### 为什么需要堆内存管理？

在嵌入式系统中，内存是宝贵的资源。我们需要：

1. **动态分配** - 运行时才知道需要多少内存
2. **内存复用** - 不同时间使用同一块内存
3. **避免浪费** - 只分配真正需要的内存
4. **简化设计** - 不需要预先规划所有内存使用

## 静态 vs 动态内存分配

### 静态内存分配

```c
/* 编译时就确定大小 */
char g_cBuffer[100];  /* 固定100字节 */
uint32_t g_ulArray[50]; /* 固定50个整数 */

void vSomeFunction(void)
{
    /* 函数内的局部变量也是静态分配（在栈上） */
    char cLocal[20];
}
```

**优点**：
- ✓ 简单，不需要管理
- ✓ 没有内存碎片
- ✓ 执行时间确定

**缺点**：
- ✗ 不灵活，必须预先知道大小
- ✗ 可能浪费内存（分配太多）
- ✗ 可能不够用（分配太少）

### 动态内存分配

```c
void vSomeFunction(void)
{
    char *pcBuffer;
    uint32_t ulSize = calculateNeededSize(); /* 运行时计算 */

    /* 动态分配 */
    pcBuffer = pvPortMalloc(ulSize);

    if(pcBuffer != NULL)
    {
        /* 使用内存 */
        useBuffer(pcBuffer);

        /* 释放内存 */
        vPortFree(pcBuffer);
    }
}
```

**优点**：
- ✓ 灵活，按需分配
- ✓ 内存可以复用
- ✓ 适应变化的需求

**缺点**：
- ✗ 需要管理分配和释放
- ✗ 可能产生内存碎片
- ✗ 执行时间不确定

## FreeRTOS的5种Heap分配器

FreeRTOS提供了5种不同的内存分配器，每种都有不同的特点：

| 分配器 | 文件 | 特点 | 适用场景 |
|--------|------|------|---------|
| **heap_1** | heap_1.c | 最简单，不支持释放 | 启动时静态分配所有内存 |
| **heap_2** | heap_2.c | 支持释放，有碎片 | 简单的动态分配 |
| **heap_3** | heap_3.c | 封装标准库malloc/free | 需要使用标准库 |
| **heap_4** | heap_4.c | 支持释放，合并相邻块 | 通用场景，推荐 |
| **heap_5** | heap_5.c | 支持多个内存区域 | 非连续内存的系统 |

### 如何选择？

1. **heap_1** - 最简单
   - 系统启动后不删除任何任务/队列
   - 所有内存都在启动时分配
   - 对确定性要求极高

2. **heap_2** - 简单的动态分配
   - 需要分配和释放
   - 分配大小比较固定
   - 可以接受一些碎片

3. **heap_4** - 推荐的通用方案
   - 需要分配和释放
   - 分配大小变化较大
   - 希望减少碎片

4. **heap_5** - 多区域内存
   - 内存不连续（如内部RAM + 外部RAM）
   - 需要跨区域管理

## 堆内存的配置

### 在FreeRTOSConfig.h中

```c
/* 堆的总大小，根据你的RAM调整 */
#define configTOTAL_HEAP_SIZE    ( ( size_t ) ( 50 * 1024 ) )  /* 50KB */

/* 启用静态分配（可选） */
#define configSUPPORT_STATIC_ALLOCATION    0

/* 内存分配失败钩子（可选） */
#define configUSE_MALLOC_FAILED_HOOK      1
```

### 内存布局

```
┌─────────────────────────────────┐ 0x20000000 (SRAM起始)
│                                 │
│   栈（Stack）- 向下增长         │
│                                 │
├─────────────────────────────────┤
│                                 │

│                                 │

│                                 │
│   堆（Heap）- 向上增长          │ ← configTOTAL_HEAP_SIZE
│                                 │

```

## 内存碎片

### 什么是内存碎片？

想象一个停车场：

```
开始：[空][空][空][空][空]

分配A：[A][空][空][空][空]
分配B：[A][B][空][空][空]
分配C：[A][B][C][空][空]

释放B：[A][空][C][空][空]  ← 碎片出现！

现在要分配一个需要2个位置的D：
虽然总共有2个空位，但不连续，无法分配！
```

### 碎片的类型

1. **内部碎片** - 分配的内存比需要的多
   ```
   需要20字节，但分配器最小分配32字节
   浪费了12字节
   ```

2. **外部碎片** - 空闲内存总量足够，但不连续
   ```
   总空闲100字节，但分成了50+50
   无法分配一个80字节的块
   ```

### 为什么heap_1没有碎片？

因为heap_1**不支持释放**！

```
分配A：[A][空][空][空][空]
分配B：[A][B][空][空][空]
分配C：[A][B][C][空][空]

（不释放，所以不会有空隙）
```

这是heap_1的一大优势，但也是它的限制。

## 重要的配置选项

在 FreeRTOSConfig.h 中：

```c
/* 堆的总大小 */
#define configTOTAL_HEAP_SIZE           ( ( size_t ) ( 50 * 1024 ) )

/* 启用静态分配 */
#define configSUPPORT_STATIC_ALLOCATION  0

/* 内存分配失败钩子 */
#define configUSE_MALLOC_FAILED_HOOK     1
```

## 总结

- **堆内存管理** = 动态管理内存的方式
- **FreeRTOS提供5种分配器**，从简单到复杂
- **heap_1最简单**，但不支持释放
- **内存碎片**是动态分配需要解决的问题
- **根据需求选择**合适的分配器
