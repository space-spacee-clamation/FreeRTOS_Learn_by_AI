# 01 - 任务概念介绍

## 什么是任务？

### 日常生活中的类比

想象一个**餐厅**的场景：

- **裸机编程**：就像只有一个服务员，他需要同时：
  - 接待顾客
  - 写菜单
  - 做菜
  - 上菜
  - 结账
  
  这个服务员会非常忙，而且顾客体验不好，因为每件事都得排队等。

- **FreeRTOS任务**：就像餐厅有多个员工：
  - 👨‍💼 一个接待员（任务1）
  - 👨‍🍳 一个厨师（任务2）
  - 👩‍💼 一个服务员（任务3）
  
  每个人专注做自己的事，虽然只有一个厨房（CPU），但通过合理安排，效率大大提高！

### 技术定义

**FreeRTOS任务**是一个独立的执行线程，具有：

- ✅ **自己的栈空间** - 就像每个人有自己的工作台
- ✅ **自己的程序计数器** - 记录执行到哪了
- ✅ **自己的CPU寄存器状态** - 保存临时数据
- ✅ **独立的优先级** - 决定谁先执行

## 任务 vs 裸机编程

### 裸机编程的特点

```c
// 裸机编程示例
int main(void)
{
    硬件初始化();
    
    while(1)  // 超级循环
    {
        处理串口();
        处理按键();
        处理显示();
        处理传感器();
        // ... 更多功能
    }
}
```

**缺点**：
- ❌ 所有功能都在一个循环里，互相影响
- ❌ 如果某个功能耗时太长，其他功能会被卡住
- ❌ 难以实现真正的"同时"处理
- ❌ 代码复杂度高，难以维护

### FreeRTOS多任务的特点

```c
// FreeRTOS多任务示例
int main(void)
{
    硬件初始化();
    
    // 创建多个独立的任务
    xTaskCreate(串口处理任务, ...);
    xTaskCreate(按键处理任务, ...);
    xTaskCreate(显示处理任务, ...);
    xTaskCreate(传感器处理任务, ...);
    
    // 启动调度器
    vTaskStartScheduler();
}

// 每个任务都是独立的函数
void 串口处理任务(void *pvParameters)
{
    for(;;)
    {
        // 只专注于串口处理
        处理串口();
        vTaskDelay(10);  // 休息一下，让其他任务运行
    }
}
```

**优点**：
- ✅ 每个任务独立，互不干扰
- ✅ 可以设置优先级，重要的事先做
- ✅ 代码结构清晰，易于理解和维护
- ✅ 可以实现"伪并行"处理

## 任务的工作原理

### 任务状态

FreeRTOS的任务有以下几种状态：

```
运行态 (Running)
    ↓
就绪态 (Ready) ← ← ← ←
    ↓                ↑
阻塞态 (Blocked) → → →
    ↓
挂起态 (Suspended)
```

1. **运行态**：任务正在使用CPU
   - 同一时刻只有一个任务处于运行态
   
2. **就绪态**：任务准备好了，等待CPU
   - "我已经准备好了，让我运行！"
   
3. **阻塞态**：任务在等待某个事件
   - "我在等延时结束/等数据/等信号量"
   
4. **挂起态**：任务被暂停了
   - "我被暂停了，除非有人恢复我"

### 上下文切换

**上下文切换**就是CPU从一个任务切换到另一个任务的过程。

**类比**：就像看书时被打断
1. 记住当前看到第几页（保存寄存器状态）
2. 把书签夹在这一页（保存程序计数器）
3. 去做另一件事（运行另一个任务）
4. 回来时，从书签位置继续（恢复寄存器状态）

**整个过程由FreeRTOS调度器自动完成，用户不需要关心！**

## 为什么需要多任务？

### 场景1：按键 + 延时

**裸机编程的问题**：
```c
// 裸机编程
while(1)
{
    if(按键按下())
    {
        处理按键();
    }
    
    // 延时1秒
    delay_ms(1000);  // ❌ 这1秒内无法检测按键！
    
    更新显示();
}
```

问题：在1秒的延时期间，按键按下了也检测不到！

**FreeRTOS的解决方案**：
```c
// 任务1：按键检测
void 按键任务(void *pvParameters)
{
    for(;;)
    {
        if(按键按下())
        {
            处理按键();
        }
        vTaskDelay(10);  // 只延时10毫秒
    }
}

// 任务2：显示更新
void 显示任务(void *pvParameters)
{
    for(;;)
    {
        更新显示();
        vTaskDelay(1000);  // 延时1秒
    }
}
```

优点：按键任务每10ms检查一次，不会错过按键！

### 场景2：多个重要性不同的任务

有些事情紧急，有些事情不紧急：

- **高优先级**：安全检测（必须立即响应）
- **中优先级**：通信处理（需要及时处理）
- **低优先级**：日志记录（有空再做）

FreeRTOS可以通过优先级设置，确保重要的任务优先执行！

## 任务的基本特性

### 1. 任务函数

```c
// 任务函数的标准格式
void vMyTask(void *pvParameters)
{
    // 可选：初始化代码（只运行一次）
    
    for(;;)  // 必须是无限循环！
    {
        // 任务主体代码
        
        // 必须有某种方式让出CPU
        vTaskDelay(100);  // 或者等待信号量、队列等
    }
    
    // 任务永远不应该到达这里！
    // 如果到达，任务会被删除
}
```

### 2. 任务优先级

- 数值越大，优先级越高
- 空闲任务优先级是0（最低）
- 相同优先级的任务轮流运行（时间片轮转）

```c
// 优先级示例
#define 优先级_安全   5  // 最高
#define 优先级_通信   3
#define 优先级_显示   2
#define 优先级_日志   1  // 最低
```

### 3. 任务栈

每个任务都有自己的栈空间：
- 用于存储局部变量
- 用于函数调用
- 用于保存上下文

**栈大小的选择**：
- 太小：栈溢出 → 系统崩溃！
- 太大：浪费内存

```c
// 栈大小设置（单位：字，32位系统中1字=4字节）
#define 任务栈大小  128  // 128 × 4 = 512字节

xTaskCreate(vMyTask, "Task", 任务栈大小, NULL, 1, NULL);
```

## 总结

### 关键要点

1. ✅ **任务**是FreeRTOS中独立的执行线程
2. ✅ **多任务**让程序结构更清晰，响应更及时
3. ✅ **调度器**自动管理任务切换，用户不需要关心
4. ✅ **优先级**决定哪个任务先执行
5. ✅ **任务函数**必须是无限循环，不能返回
