# 05 - 调度器启动指南

## vTaskStartScheduler() 函数

### 函数原型

```c
void vTaskStartScheduler(void);
```

### 特点

- **没有参数**
- **没有返回值**（成功时永远不返回！）
- 是FreeRTOS程序的关键一步

---

## 启动调度器之前

### 你的代码应该做什么

```c
int main(void)
{
    // 1. 硬件初始化
    硬件初始化();
    
    // 2. 创建系统对象（队列、信号量等）
    xQueue = xQueueCreate(...);
    xSemaphore = xSemaphoreCreateMutex();
    
    // 3. 创建任务
    xTaskCreate(vTask1, ...);
    xTaskCreate(vTask2, ...);
    xTaskCreate(vTask3, ...);
    
    // 4. 启动调度器（关键！）
    vTaskStartScheduler();
    
    // 5. 如果到这里，说明启动失败了
    for(;;)
    {
        // 处理错误
    }
}
```

---

## vTaskStartScheduler() 做了什么？

### 内部流程

```
调用 vTaskStartScheduler()
    ↓
1. 创建空闲任务（IDLE任务）
    ↓
2. 如果启用软件定时器，创建定时器服务任务
    ↓
3. 配置系统时钟节拍中断
    ↓
4. 启动第一个任务（通常是最高优先级的就绪任务）
    ↓
5. 调度器开始工作！（永远不返回）
```

---

### 详细说明

#### 1. 创建空闲任务

**空闲任务**：
- 优先级：0（最低）
- 作用：当没有其他任务运行时运行
- 必须存在：FreeRTOS不能没有空闲任务

```c
// FreeRTOS内部大概是这样创建空闲任务的
xTaskCreate(
    prvIdleTask,          // 空闲任务函数
    "IDLE",               // 任务名称
    configIDLE_TASK_STACK_SIZE,  // 栈大小
    NULL,                 // 参数
    0,                    // 优先级（最低）
    NULL                  // 句柄
);
```

---

#### 2. 创建定时器任务（如果启用）

```c
// 如果在 FreeRTOSConfig.h 中启用了软件定时器
#define configUSE_TIMERS    1

// FreeRTOS会自动创建定时器服务任务
xTaskCreate(
    prvTimerTask,
    "Tmr Svc",
    configTIMER_TASK_STACK_DEPTH,
    NULL,
    configTIMER_TASK_PRIORITY,
    NULL
);
```

---

#### 3. 配置系统时钟节拍

FreeRTOS需要一个定期的中断来驱动调度器。

**对于Cortex-M3/M4**：
- 使用SysTick定时器
- 配置 `configTICK_RATE_HZ` 频率

---

#### 4. 启动第一个任务

调度器选择优先级最高的就绪任务开始运行。

---

## 启动成功 vs 启动失败

### 成功的情况

```c
vTaskStartScheduler();
// 调度器启动成功！
// 永远不会到达这里！
```

**现象**：
- ✅ 任务开始运行
- ✅ 串口有输出
- ✅ 系统正常工作

---

### 失败的情况

```c
vTaskStartScheduler();
// 调度器启动失败！
// 会到达这里
```

**现象**：
- ❌ 没有任务运行
- ❌ 系统"卡住"
- ❌ 程序执行到了 `vTaskStartScheduler()` 之后

---

## 启动失败的常见原因

### 原因1：内存不足，无法创建空闲任务

**最常见的原因！**

```c
// 问题：堆内存太小
#define configTOTAL_HEAP_SIZE    (1024)  // 只有1KB，太小了！

vTaskStartScheduler();  // ❌ 无法创建空闲任务，失败！
```

**解决方法**：
```c
// 增加堆内存大小
#define configTOTAL_HEAP_SIZE    (10 * 1024)  // 10KB，应该够了
```

---

### 原因2：使用了错误的内存分配方案

```c
// 问题：某些内存分配方案不适合
#define configUSE_MALLOC_FAILED_HOOK    0

// 解决：检查使用的heap_x.c
// 推荐使用 heap_2.c 或 heap_4.c
```

---

### 原因3：中断优先级配置错误

对于Cortex-M，需要正确配置中断优先级。

```c
// 在 FreeRTOSConfig.h 中
#define configPRIO_BITS    4  // 根据实际情况设置
#define configLIBRARY_LOWEST_INTERRUPT_PRIORITY    15
#define configLIBRARY_MAX_SYSCALL_INTERRUPT_PRIORITY    5
```

---

## 启动前检查清单

在调用 `vTaskStartScheduler()` 之前，检查：

- [ ] 硬件已经初始化
- [ ] 至少创建了一个任务
- [ ] 堆内存大小足够（推荐至少5KB）
- [ ] FreeRTOSConfig.h 配置正确
- [ ] 中断优先级配置正确（如果使用中断）
- [ ] 有错误处理代码（如果启动失败）

## 完整示例

```c
#include "FreeRTOS.h"
#include "task.h"

// 任务函数
void vMyTask(void *pvParameters)
{
    (void)pvParameters;
    
    for(;;)
    {
        // 任务代码
        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}

int main(void)
{
    BaseType_t xResult;
    
    // 1. 硬件初始化
    硬件初始化();
    
    // 2. 创建任务
    xResult = xTaskCreate(
        vMyTask,
        "MyTask",
        128,
        NULL,
        1,
        NULL
    );
    
    if(xResult != pdPASS)
    {
        // 任务创建失败
        处理错误();
        for(;;);
    }
    
    // 3. 准备启动调度器
    输出信息("准备启动调度器...\r\n");
    
    // 4. 启动调度器
    vTaskStartScheduler();
    
    // 5. 如果到这里，说明启动失败！
    输出错误("错误：调度器启动失败！\r\n");
    输出错误("可能原因：内存不足\r\n");
    
    for(;;)
    {
        // 死循环，让用户知道出错了
    }
    
    return 0;
}
```

## 调试技巧

### 技巧1：添加启动钩子

```c
// 在 FreeRTOSConfig.h 中启用
#define configUSE_IDLE_HOOK    1

// 实现空闲钩子
void vApplicationIdleHook(void)
{
    // 如果进入这里，说明调度器启动成功了！
    LED_ON();  // 点亮LED表示成功
}
```

---

### 技巧2：检查内存

```c
// 启动前检查空闲堆大小
void vPrintFreeHeapSize(void)
{
    size_t xFreeHeapSize = xPortGetFreeHeapSize();
    输出数字("空闲堆内存：", xFreeHeapSize);
    输出字符串(" 字节\r\n");
}

int main(void)
{
    // ... 创建任务 ...
    
    // 检查堆内存
    vPrintFreeHeapSize();
    
    // 启动调度器
    vTaskStartScheduler();
}
```

---

### 技巧3：启用MALLOC失败钩子

```c
// 在 FreeRTOSConfig.h 中启用
#define configUSE_MALLOC_FAILED_HOOK    1

// 实现钩子
void vApplicationMallocFailedHook(void)
{
    // 内存分配失败！
    输出错误("内存分配失败！\r\n");
    for(;;);
}
```

## 总结

### 关键点

1. ✅ `vTaskStartScheduler()` 启动FreeRTOS调度器
2. ✅ 成功时**永远不返回**
3. ✅ 失败时会返回，需要处理错误
4. ✅ 启动前要创建好任务
5. ✅ 最常见失败原因：内存不足
6. ✅ 确保 `configTOTAL_HEAP_SIZE` 足够大
