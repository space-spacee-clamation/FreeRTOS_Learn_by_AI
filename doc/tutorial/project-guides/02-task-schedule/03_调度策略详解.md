# 调度策略详解

## FreeRTOS的调度策略

FreeRTOS使用两种主要的调度策略：

1. **抢占式调度**（基于优先级）
2. **时间片轮转**（相同优先级之间）

让我们详细了解这两种策略。

## 1. 抢占式调度

### 什么是抢占式调度？

**抢占式调度** 意味着高优先级任务可以在任何时候抢占低优先级任务的CPU使用权。

### 生活中的类比

想象一架飞机正在等待降落：

- **紧急情况**（如医疗急救）：可以插队，优先降落
- **普通航班**：按顺序等待
- **小飞机**：等所有大型飞机降落后才能降落

在FreeRTOS中：
- **高优先级任务** = 紧急航班
- **低优先级任务** = 普通航班
- **抢占** = 紧急航班插队

### 抢占的时机

抢占可能发生在以下情况：

1. **高优先级任务就绪**
2. **系统时钟节拍中断**
3. **任务调用阻塞函数**

### 抢占示例

假设有3个任务：

| 任务 | 优先级 | 状态 |
|-----|-------|------|
| TaskA | 3（高） | 阻塞中 |
| TaskB | 2（中） | 运行中 |
| TaskC | 1（低） | 就绪 |

**场景1：高优先级任务就绪**

```
时间轴 →
[TaskB运行] → [TaskA就绪！] → [TaskA抢占运行] → [TaskA阻塞] → [TaskB继续] → ...
                     ↑
                抢占发生！
```

**场景2：任务阻塞**

```
时间轴 →
[TaskB运行] → [TaskB调用vTaskDelay()] → [TaskC运行] → [TaskB延时结束] → [TaskB抢占] → ...
                                               ↑
                                          TaskC获得机会
```

### 抢占式调度的特点

✅ **优点**：
- 高优先级任务响应快
- 实时性好
- 适合时间关键的应用

⚠️ **注意事项**：
- 需要处理任务同步问题
- 可能发生优先级反转
- 低优先级任务可能饿死

## 2. 时间片轮转

### 什么是时间片轮转？

**时间片轮转** 是指相同优先级的任务之间平分CPU时间。

### 生活中的类比

想象几个朋友共用一台游戏机：

- 每个人玩10分钟（时间片）
- 10分钟到了，换下一个人
- 这样大家都有机会玩

在FreeRTOS中：
- **相同优先级任务** = 玩游戏的朋友
- **时间片** = 10分钟
- **轮转** = 轮流玩

### 时间片的大小

时间片大小由系统时钟节拍决定：

```c
// 在 FreeRTOSConfig.h 中配置
#define configTICK_RATE_HZ  1000  // 每秒1000个节拍
```

如果 `configTICK_RATE_HZ = 1000`，那么：
- 1个时钟节拍 = 1毫秒
- 默认时间片 = 1个节拍 = 1毫秒

### 时间片轮转示例

假设有3个相同优先级的任务：

| 任务 | 优先级 |
|-----|-------|
| Task1 | 2 |
| Task2 | 2 |
| Task3 | 2 |

**运行顺序**：

```
时间轴 →
[Task1] → [Task2] → [Task3] → [Task1] → [Task2] → [Task3] → ...
  1ms      1ms      1ms      1ms      1ms      1ms
```

每个任务运行1个时间片，然后切换到下一个。

### 时间片轮转的特点

✅ **优点**：
- 公平分配CPU时间
- 相同优先级任务都有机会运行
- 实现简单

⚠️ **注意事项**：
- 频繁切换会有开销
- 不适合需要连续运行的任务
- 响应时间取决于时间片大小

## 3. 调度器的工作流程

### 完整的调度决策过程

调度器在每次需要选择任务时，按以下步骤决策：

```
1. 收集所有就绪任务
   ↓
2. 找出优先级最高的任务组
   ↓
3. 如果该组只有1个任务
   ├─ 是 → 直接选择这个任务
   └─ 否 → 在该组内使用时间片轮转
   ↓
4. 切换到选中的任务
```

### 示例1：混合优先级

就绪任务列表：

| 任务 | 优先级 |
|-----|-------|
| TaskA | 3 |
| TaskB | 2 |
| TaskC | 2 |
| TaskD | 1 |

**调度决策**：
1. 最高优先级是3
2. 只有TaskA一个任务
3. 选择TaskA运行

### 示例2：相同优先级

就绪任务列表：

| 任务 | 优先级 |
|-----|-------|
| Task1 | 2 |
| Task2 | 2 |
| Task3 | 2 |

**调度决策**：
1. 最高优先级是2
2. 有3个任务
3. 使用时间片轮转
4. 选择等待时间最长的任务

## 4. 重要配置选项

### 启用/禁用抢占

```c
// 在 FreeRTOSConfig.h 中

// 启用抢占式调度（1=启用，0=禁用）
#define configUSE_PREEMPTION        1

// 如果禁用抢占，就变成协作式调度
```

### 启用/禁用时间片轮转

```c
// 启用时间片轮转（1=启用，0=禁用）
#define configUSE_TIME_SLICING      1

// 如果禁用，相同优先级任务不会自动切换
```

### 时钟节拍频率

```c
// 系统时钟节拍频率（Hz）
#define configTICK_RATE_HZ          1000

// 常用值：1000（1ms）、500（2ms）、100（10ms）
```

## 5. 调度策略的选择

### 什么时候用抢占式调度？

✅ **推荐使用抢占式的场景**：
- 实时性要求高的系统
- 需要快速响应的应用
- 有时间关键任务的项目
- 大多数嵌入式系统

### 什么时候可以不用抢占式？

⚠️ **可以考虑协作式的场景**：
- 非常简单的系统
- 任务之间不需要快速响应
- 对实时性要求不高
- 想简化同步问题

### 时间片轮转的使用建议

✅ **推荐启用时间片轮转**：
- 有多个相同优先级任务
- 需要公平分配CPU时间
- 大多数情况都应该启用

## 6. 本项目的调度演示

在示例项目中，你可以观察到：

### 抢占式调度的演示

```
[High] 高优先级任务运行  ← 最高优先级，最先运行
[Medium] 中优先级任务运行 ← 只在High阻塞时运行
[Low] 低优先级任务运行    ← 只在High和Medium都阻塞时运行
```

### 如果High不阻塞会怎样？

修改代码，让High任务不调用vTaskDelay()：

```c
void vHighPriorityTask(void *pvParameters)
{
    for(;;)
    {
        // 不调用vTaskDelay()，永远不阻塞
        做一些工作();
    }
}
```

**结果**：
- High任务永远运行
- Medium和Low任务永远得不到运行机会
- 这就是"饿死"现象

## 总结

- **抢占式调度**：高优先级抢占低优先级
- **时间片轮转**：相同优先级任务平分CPU
- **configUSE_PREEMPTION**：控制是否启用抢占
- **configUSE_TIME_SLICING**：控制是否启用时间片轮转
- **configTICK_RATE_HZ**：控制系统时钟节拍频率
