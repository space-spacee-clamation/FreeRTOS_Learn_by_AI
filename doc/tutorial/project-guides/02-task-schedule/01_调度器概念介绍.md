# 调度器概念介绍

## 什么是调度器？

### 生活中的类比

想象一个工厂有多个工人（任务）和一个工头（调度器）：

- **工人**：负责具体的工作（比如打包、质检、搬运）
- **工头**：决定哪个工人在什么时候使用机器（CPU）
- **机器**：只有一台，同一时间只能一个工人使用

在FreeRTOS中：
- **任务** = 工人
- **调度器** = 工头
- **CPU** = 机器

### 调度器的定义

**调度器（Scheduler）** 是FreeRTOS内核的核心组件，负责决定：
- 哪个任务应该在当前运行
- 什么时候切换任务
- 如何分配CPU时间给各个任务

## 为什么需要调度器？

### 裸机编程的问题

在没有操作系统的裸机编程中，程序通常是这样的：

```c
int main(void)
{
    初始化硬件();
    
    while(1)  // 超级循环
    {
        处理传感器();
        处理通信();
        处理显示();
        处理按键();
    }
}
```

**问题**：
1. 如果传感器处理很慢，通信会被延迟
2. 如果通信处理很久，显示会卡顿
3. 所有功能必须按顺序执行，无法灵活调度

### 调度器的优势

使用FreeRTOS调度器后：

```c
void 传感器任务(void *p)
{
    while(1) {
        读取传感器();
        vTaskDelay(10);  // 阻塞，让其他任务运行
    }
}

void 通信任务(void *p)
{
    while(1) {
        处理通信();
        vTaskDelay(5);   // 阻塞，让其他任务运行
    }
}

void 显示任务(void *p)
{
    while(1) {
        更新显示();
        vTaskDelay(20);  // 阻塞，让其他任务运行
    }
}
```

**优势**：
1. 每个任务独立运行，互不干扰
2. 调度器根据优先级决定谁先运行
3. 任务可以阻塞，不浪费CPU时间
4. 系统响应更快，更灵活

## 两种调度方式

### 1. 协作式调度（Cooperative Scheduling）

**特点**：任务主动放弃CPU控制权

```
任务A运行 → 任务A主动放弃 → 任务B运行 → 任务B主动放弃 → ...
```

**优点**：
- 实现简单
- 不需要处理复杂的同步问题

**缺点**：
- 如果一个任务不主动放弃，其他任务永远无法运行
- 响应时间不可预测

### 2. 抢占式调度（Preemptive Scheduling）

**特点**：调度器可以强制暂停当前任务，切换到其他任务

```
任务A运行 → [调度器介入] → 任务B运行 → [调度器介入] → 任务A运行 → ...
```

**优点**：
- 系统响应快
- 高优先级任务可以立即执行
- 实时性好

**缺点**：
- 需要处理任务同步问题
- 实现相对复杂

## FreeRTOS的调度器

FreeRTOS默认使用**抢占式调度器**，但也可以配置为协作式。

### 抢占式调度的工作原理

1. **高优先级抢占**：如果高优先级任务就绪，立即抢占CPU
2. **时间片轮转**：相同优先级的任务之间平分CPU时间
3. **阻塞机制**：任务可以主动阻塞，让出CPU

### 调度器的决策依据

调度器在决定切换任务时，会考虑：

1. **任务状态**：就绪、运行、阻塞、挂起
2. **任务优先级**：高优先级优先
3. **等待时间**：相同优先级时，等待最久的先运行

## 任务状态

FreeRTOS中的任务有4种基本状态：

### 1. 运行态（Running）

- 任务正在使用CPU
- 同一时间只有一个任务处于运行态

### 2. 就绪态（Ready）

- 任务可以运行，但还没轮到
- 等待调度器选中

### 3. 阻塞态（Blocked）

- 任务在等待某个事件（如延时、信号量、队列等）
- 暂时不参与调度

### 4. 挂起态（Suspended）

- 任务被暂停，不参与调度
- 需要手动恢复

## 状态转换图

```
        创建
          ↓
   ┌──→ 就绪 ←────┐
   |      ↑        |
   |      | 抢占   |
阻塞 ←── 运行 →─ 挂起
   |               ↑
   └──── 恢复 ────┘
```

## 调度器的启动

调用 `vTaskStartScheduler()` 后：

1. 创建空闲任务（优先级0）
2. 如果启用了定时器，创建定时器任务
3. 启动系统时钟节拍中断
4. 选择最高优先级的就绪任务开始运行
5. 调度器开始工作

## 重要配置选项

在 FreeRTOSConfig.h 中：

```c
// 启用抢占式调度（1=抢占，0=协作）
#define configUSE_PREEMPTION        1

// 启用时间片轮转（相同优先级任务之间）
#define configUSE_TIME_SLICING      1

// 系统时钟节拍频率（Hz）
#define configTICK_RATE_HZ          1000

// 最大优先级数
#define configMAX_PRIORITIES        6
```

## 总结

- **调度器**：决定哪个任务运行的"工头"
- **抢占式调度**：高优先级任务可以立即抢占CPU
- **任务状态**：运行、就绪、阻塞、挂起
- **FreeRTOS**：默认使用抢占式调度器
