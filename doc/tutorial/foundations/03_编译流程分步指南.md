# 编译流程分步指南

## 📚 概述

本文档通过实际操作，一步步展示从源代码到可执行文件的完整编译过程。

## 🎯 学习目标

完成本指南后，你将理解：
1. 预处理（Preprocessing）
2. 编译（Compilation）
3. 汇编（Assembly）
4. 链接（Linking）
5. 各阶段生成的文件格式

## 📋 前置准备

进入项目目录，先清理之前的编译产物：

```bash
cd projects/stm32-learning/01-getting-started/04-semaphore
make clean
```

## 🔍 分步编译详解

我们将手动执行每个编译步骤，看看会发生什么。

### 第一步：设置环境变量

首先定义一些有用的变量：

```bash
# 工具链
CROSS_COMPILE=arm-none-eabi-
CC=${CROSS_COMPILE}gcc

# FreeRTOS路径（相对于项目目录）
FREERTOS_ROOT=../../../../FreeRTOS/FreeRTOS
FREERTOS_SRC=${FREERTOS_ROOT}/Source
FREERTOS_INC=${FREERTOS_SRC}/include
FREERTOS_PORT=${FREERTOS_SRC}/portable/GCC/ARM_CM3

# 编译器标志
CFLAGS="-mcpu=cortex-m3 -mthumb -Og -Wall -g"
CFLAGS="$CFLAGS -I./include -I${FREERTOS_INC} -I${FREERTOS_PORT}"
CFLAGS="$CFLAGS -DPLATFORM_LM3S6965"
```

### 第二步：预处理（Preprocessing）

**预处理**处理 `#include`、`#define` 等预处理指令。

```bash
# 预处理 main.c，生成 .i 文件
${CC} ${CFLAGS} -E src/main.c -o src/main.i
```

**查看预处理后的文件**：
```bash
# 查看文件开头
head -50 src/main.i

# 查看文件大小
ls -lh src/main.i
```

**预处理做了什么**：
1. ✅ 展开所有 `#include` 头文件
2. ✅ 替换所有 `#define` 宏定义
3. ✅ 处理条件编译 `#if`、`#ifdef` 等
4. ✅ 删除所有注释
5. ✅ 添加行号标记

### 第三步：编译（Compilation）

**编译**将预处理后的C代码转换成汇编代码。

```bash
# 编译成汇编，生成 .s 文件
${CC} ${CFLAGS} -S src/main.i -o src/main.s
```

**查看汇编代码**：
```bash
# 查看汇编文件
cat src/main.s
```

**编译做了什么**：
1. ✅ 语法分析和语义分析
2. ✅ 优化代码
3. ✅ 生成汇编指令

**汇编代码示例**：
```assembly
    .syntax unified
    .cpu cortex-m3
    .fpu softvfp
    .thumb
    .file   "main.c"
    .text
    .align  2
    .global vTask1
    .thumb
    .thumb_func
    .type   vTask1, %function
vTask1:
    @ 函数体的汇编指令...
```

### 第四步：汇编（Assembly）

**汇编**将汇编代码转换成机器码（目标文件）。

```bash
# 汇编成目标文件，生成 .o 文件
${CC} ${CFLAGS} -c src/main.s -o src/main.o
```

**或者直接从 .c 编译到 .o**（跳过中间步骤）：
```bash
${CC} ${CFLAGS} -c src/main.c -o src/main.o
```

**查看目标文件信息**：
```bash
# 查看文件类型
file src/main.o

# 查看符号表
${CROSS_COMPILE}nm src/main.o

# 查看段信息
${CROSS_COMPILE}objdump -h src/main.o
```

**汇编做了什么**：
1. ✅ 将汇编指令转换成机器码
2. ✅ 创建符号表
3. ✅ 生成重定位信息
4. ✅ 组织成ELF文件格式

### 第五步：编译其他文件

同样的步骤编译其他源文件：

```bash
# 编译启动文件
${CC} ${CFLAGS} -c startup_lm3s6965.c -o startup_lm3s6965.o

# 编译FreeRTOS任务
${CC} ${CFLAGS} -c ${FREERTOS_SRC}/tasks.c -o tasks.o

# 编译FreeRTOS队列
${CC} ${CFLAGS} -c ${FREERTOS_SRC}/queue.c -o queue.o

# 编译FreeRTOS链表
${CC} ${CFLAGS} -c ${FREERTOS_SRC}/list.c -o list.o

# 编译FreeRTOS定时器
${CC} ${CFLAGS} -c ${FREERTOS_SRC}/timers.c -o timers.o

# 编译移植层
${CC} ${CFLAGS} -c ${FREERTOS_PORT}/port.c -o port.o

# 编译内存管理
${CC} ${CFLAGS} -c ${FREERTOS_SRC}/portable/MemMang/heap_4.c -o heap_4.o
```

**检查所有生成的目标文件**：
```bash
ls -lh *.o src/*.o
```

### 第六步：链接（Linking）

**链接**将所有目标文件组合成一个可执行文件。

```bash
# 链接所有目标文件
LDFLAGS="-T LM3S6965.ld -nostdlib -Wl,-Map=semaphore-LM3S6965.map"

${CC} ${CFLAGS} \
    src/main.o \
    startup_lm3s6965.o \
    tasks.o \
    queue.o \
    list.o \
    timers.o \
    port.o \
    heap_4.o \
    ${LDFLAGS} \
    -o semaphore-LM3S6965.elf
```

**查看链接生成的文件**：
```bash
# 查看ELF文件信息
file semaphore-LM3S6965.elf

# 查看大小
${CROSS_COMPILE}size semaphore-LM3S6965.elf

# 查看符号表
${CROSS_COMPILE}nm semaphore-LM3S6965.elf | head -30

# 查看链接映射文件
cat semaphore-LM3S6965.map
```

**链接做了什么**：
1. ✅ 符号解析（函数调用、变量引用）
2. ✅ 地址分配（代码段、数据段、BSS段）
3. ✅ 重定位（修正地址引用）
4. ✅ 生成最终的可执行文件

### 第七步：生成二进制格式

从ELF文件生成BIN和HEX文件：

```bash
# 生成 BIN 文件（纯机器码）
${CROSS_COMPILE}objcopy -O binary semaphore-LM3S6965.elf semaphore-LM3S6965.bin

# 生成 HEX 文件（Intel HEX格式）
${CROSS_COMPILE}objcopy -O ihex semaphore-LM3S6965.elf semaphore-LM3S6965.hex
```

**查看生成的文件**：
```bash
ls -lh semaphore-LM3S6965.*

# 查看BIN文件（纯二进制，用hexdump看）
hexdump -C semaphore-LM3S6965.bin | head -20

# 查看HEX文件（文本格式）
head -20 semaphore-LM3S6965.hex
```

**各格式的区别**：

| 格式 | 特点 | 用途 |
|------|------|------|
| ELF | 包含调试信息、符号表 | 调试、开发阶段 |
| BIN | 纯机器码，最小 | 烧录到Flash |
| HEX | 文本格式，带地址 | 某些烧录工具 |

## 📊 完整流程图

```
C源代码 (.c)
    ↓
[预处理] -E
    ↓
预处理文件 (.i)
    ↓
[编译] -S
    ↓
汇编代码 (.s)
    ↓
[汇编] -c
    ↓
目标文件 (.o)
    ↓
[链接]
    ↓
可执行文件 (.elf)
    ↓
[格式转换] objcopy
    ↓
二进制文件 (.bin, .hex)
```

## 🎮 使用Makefile简化

当然，日常开发中我们不需要手动执行这么多步骤，Makefile会自动处理：

```bash
# 一条命令完成所有步骤
make all-lm3s6965

# 查看Makefile实际执行的命令
make --just-print all-lm3s6965
```

## 🔧 常用调试工具

### 1. 查看符号表
```bash
# 查看目标文件的符号
${CROSS_COMPILE}nm src/main.o

# 查看可执行文件的符号
${CROSS_COMPILE}nm semaphore-LM3S6965.elf
```

### 2. 反汇编
```bash
# 反汇编目标文件
${CROSS_COMPILE}objdump -d src/main.o

# 反汇编可执行文件
${CROSS_COMPILE}objdump -d semaphore-LM3S6965.elf | head -50
```

### 3. 查看段信息
```bash
# 查看各段大小
${CROSS_COMPILE}size semaphore-LM3S6965.elf

# 详细段信息
${CROSS_COMPILE}objdump -h semaphore-LM3S6965.elf
```

### 4. 查看文件头
```bash
${CROSS_COMPILE}readelf -h semaphore-LM3S6965.elf
```

## 💡 理解输出

### size命令输出解释

```
$ arm-none-eabi-size semaphore-lm3s6965.elf
   text	   data	    bss	    dec	    hex	filename
   8220	      8	  51576	  59804	   e99c	semaphore-lm3s6965.elf
```

| 段 | 含义 | 存储内容 |
|----|------|---------|
| text | 代码段 | 程序指令（只读） |
| data | 数据段 | 初始化的全局变量 |
| bss | BSS段 | 未初始化的全局变量（零初始化） |
| dec | 总计 | text + data + bss |
| hex | 十六进制 | dec的十六进制表示 |
