## 为什么需要环境验证

在开始学习 FreeRTOS 之前，确保开发环境正确配置至关重要。环境验证项目的主要目的是：

1. **提前发现问题** - 在编写复杂代码前验证基础工具链
2. **建立信心** - 确认所有必要的工具都能正常工作
3. **节省时间** - 避免在后续学习中因环境问题而受阻
4. **熟悉流程** - 了解从编译到运行的完整开发流程

环境验证是整个学习过程的第一步，也是最关键的一步。

---

## 验证成功意味着什么

成功运行环境验证项目，表明以下组件都已正确安装和配置：

### ✅ 交叉编译器可用
- `arm-none-eabi-gcc` 工具链已安装
- 可以正确编译 ARM Cortex-M 架构的代码
- 链接器能够生成可执行文件

### ✅ QEMU 可用
- `qemu-system-arm` 已安装
- 支持 STM32 开发板仿真
- 串口输出能够正常显示

### ✅ 构建系统可用
- Make 工具正常工作
- 项目依赖关系正确解析
- 清理和重建功能正常
- 支持多平台编译

### ✅ FreeRTOS 源码可用
- FreeRTOS 内核源码已正确获取
- 头文件路径配置正确
- 能够正确链接 FreeRTOS 库

## 支持的平台

环境验证项目现在支持多个平台，你可以根据需要选择：

| 平台 | QEMU机器类型 | 说明 | 推荐度 |
|------|-------------|------|--------|
| **LM3S6965** | `lm3s6965evb` | FreeRTOS官方QEMU演示项目 | ⭐⭐⭐⭐⭐ |
| **STM32F103** | `olimex-stm32-p103` | 基于STM32F103RBT6的开发板 | ⭐⭐⭐⭐⭐ |

**建议**：初学者可以从LM3S6965开始，因为它是FreeRTOS官方支持的QEMU平台；如果你对STM32F103更熟悉，也可以直接使用STM32F103平台。

## 如何运行测试项目

环境验证项目位于 `projects/stm32-learning/01-getting-started/00-env-verify/` 目录下。

### 步骤 1：进入项目目录

```bash
cd projects/stm32-learning/01-getting-started/00-env-verify
```

### 步骤 2：查看可用目标

```bash
make help
```

这会显示所有可用的编译和运行目标。

### 步骤 3：选择平台并编译

**选项 A：编译 LM3S6965 版本**
```bash
make all-lm3s6965
```

**选项 B：编译 STM32F103 版本** ⭐
```bash
make all-stm32f103
```

这个命令会：
- 编译 C 源代码文件
- 编译汇编启动文件
- 链接生成 ELF 可执行文件
- 生成二进制文件（.bin）和 Intel Hex 文件（.hex）

### 步骤 4：在 QEMU 中运行

**选项 A：运行 LM3S6965**
```bash
make run-lm3s6965
```

**选项 B：运行 STM32F103** ⭐
```bash
make run-stm32f103
```

这个命令会：
- 启动 QEMU 仿真器
- 加载编译好的程序
- 开始执行并显示输出

### 步骤 5：退出 QEMU

要退出 QEMU，请按以下组合键：
1. 按下 `Ctrl + A`
2. 松开后按下 `X`

### 其他有用的命令

```bash
# 清理所有平台的编译产物
make clean

# 启动调试模式（等待 GDB 连接）- LM3S6965
make debug-lm3s6965

# 启动调试模式（等待 GDB 连接）- STM32F103
make debug-stm32f103
```

## 常见问题排查

### 问题 1：arm-none-eabi-gcc: not found

**错误信息：**
```
make: arm-none-eabi-gcc: Command not found
```

**原因：** 交叉编译工具链未安装或不在 PATH 中。

**解决方法：**
```bash
# Ubuntu/Debian 系统
sudo apt update
sudo apt install gcc-arm-none-eabi

# 或者重新运行项目初始化脚本
cd /home/space/FreeRTOSDemo
./init.sh
```

**验证安装：**
```bash
arm-none-eabi-gcc --version
```

---

### 问题 2：找不到 FreeRTOS.h

**错误信息：**
```
fatal error: FreeRTOS.h: No such file or directory
```

**原因：** FreeRTOS 源码未正确获取或路径配置错误。

**解决方法：**
1. 确认项目根目录下存在 `FreeRTOS` 目录
2. 如果不存在，运行初始化脚本：
   ```bash
   cd /home/space/FreeRTOSDemo
   ./init.sh
   ```
3. 检查目录结构：
   ```bash
   ls -la FreeRTOS/FreeRTOS/Source/include/
   ```
   应该能看到 `FreeRTOS.h` 文件。

---

### 问题 3：QEMU 无输出

**症状：** 运行 `make run-xxx` 后，QEMU 启动但没有任何输出。

**可能原因和解决方法：**

#### 检查 QEMU 安装
```bash
qemu-system-arm --version
```

如果未安装：
```bash
sudo apt install qemu-system-arm
```

#### 检查 QEMU 对相关机器的支持
```bash
# 检查STM32相关支持
qemu-system-arm -machine help | grep -E "stm32|lm3s"
```

应该能看到 `lm3s6965evb` 和 `olimex-stm32-p103` 等选项。

#### 尝试详细输出模式
修改 Makefile 中的 QEMU 命令，添加 `-d guest_errors` 参数查看详细信息。

---

### 问题 4：链接错误 - undefined reference

**错误信息：**
```
undefined reference to `vTaskStartScheduler'
undefined reference to `xTaskCreate'
```

**原因：** FreeRTOS 源文件未正确编译或链接。

**解决方法：**
1. 检查 Makefile 中的 FreeRTOS 源文件路径
2. 确认 `FreeRTOS/FreeRTOS/Source/` 目录下存在所需的 .c 文件
3. 运行 `make clean` 后重新编译：
   ```bash
   make clean
   make all-stm32f103  # 或者 make all-lm3s6965
   ```

---

### 问题 5：权限错误

**错误信息：**
```
make: execvp: arm-none-eabi-gcc: Permission denied
```

**原因：** 工具链文件权限不正确。

**解决方法：**
```bash
# 检查工具链安装位置
which arm-none-eabi-gcc

# 修复权限（假设安装在 /usr/bin/）
sudo chmod +x /usr/bin/arm-none-eabi-*
```

---

### 问题 6：QEMU 立即退出

**症状：** `make run-xxx` 后 QEMU 立即退出，没有错误信息。

**诊断步骤：**
1. 检查 ELF 文件是否有效：
   ```bash
   file env-verify-stm32f103.elf  # 或者 env-verify-lm3s6965.elf
   ```
   应该显示 "ARM executable, 32-bit LSB"

2. 手动运行 QEMU 查看错误：
   ```bash
   # 对于STM32F103
   qemu-system-arm -machine olimex-stm32-p103 -nographic -kernel env-verify-stm32f103.elf
   
   # 对于LM3S6965
   qemu-system-arm -machine lm3s6965evb -nographic -kernel env-verify-lm3s6965.elf
   ```

3. 检查链接脚本和启动文件是否匹配。

---

### 问题 7：编译错误：make: *** No rule to make target

**错误信息：**
```
make: *** No rule to make target 'all'.  Stop.
```

**原因：** 使用了错误的目标名称。

**解决方法：**
- 确认使用了正确的平台后缀：
  - `make all-lm3s6965` （不是 `make all`）
  - `make all-stm32f103` （不是 `make all`）
- 运行 `make help` 查看所有可用目标
