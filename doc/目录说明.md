# doc 目录说明

## 目录结构与用途

### 1. doc/experiences/ - 经验沉淀目录
按技术分类存储开发过程中的经验总结、问题排查记录。

**分类结构：**
- `stm32/` - STM32相关开发经验
- `freertos/` - FreeRTOS相关开发经验
- `build-system/` - 编译构建系统相关经验
- `qemu/` - QEMU模拟器相关经验

**维护原则：**
- 及时删除过时、错误内容
- 保持文档准确性
- 新经验按照技术分类存入对应子目录
- 每个经验文档聚焦一个问题或知识点

### 2. doc/tutorial/ - 教程文档目录
存放学习项目的教程文档，对应`projects/`目录中的学习项目。

结构为 `tutorial/教程名称/`，包含对应教程的拆分文档和索引。

当前已有：
- `Mastering-the-FreeRTOS-Kernel/` - 《Mastering the FreeRTOS Real-Time Kernel》书籍拆分文档

### 3. 文档与代码分离原则

本项目遵循**文档与代码分离**的组织方式：
- **文档** 统一存放在 `doc/` 目录下，按上述分类管理
- **代码** 存放在 `projects/` 目录下，按项目组织

这样的好处：
- 文档可以独立维护更新
- 经验沉淀便于检索和复用
- 学习项目代码与对应教程清晰分开
- 便于AI辅助开发和维护
