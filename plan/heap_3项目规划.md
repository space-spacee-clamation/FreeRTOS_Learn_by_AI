# heap_3 内存分配器学习项目规划

## 项目概述

本项目将创建一个FreeRTOS heap_3内存分配器的学习示例项目，演示标准库malloc/free包装器的使用方法。

## heap_3 核心特点

### 工作原理
- **包装器设计**：简单封装标准C库的`malloc()`和`free()`函数
- **线程安全**：通过`vTaskSuspendAll()`和`xTaskResumeAll()`保证线程安全
- **依赖链接器**：需要链接器配置堆内存区域

### 优势
1. 实现最简单，代码量最少
2. 可以利用标准库的成熟实现
3. 支持完全的内存分配和释放
4. 与标准C库兼容

### 劣势
1. 不可确定的执行时间
2. 效率相对较低
3. 内存碎片问题取决于标准库实现
4. 不提供内存统计信息（xPortGetFreeHeapSize()等不可用）
5. 依赖链接器配置

## 主程序设计要点

### 学习演示内容
1. **基础分配释放**：演示malloc和free的基本使用
2. **多任务并发**：展示多个任务同时使用内存分配
3. **与heap_1/2对比**：突出显示heap_3的特殊性
4. **注意事项说明**：强调执行时间不确定性等问题

### 演示步骤
1. 初始化和介绍heap_3特点
2. 单个任务的内存分配和释放
3. 多个任务并发分配内存
4. 演示不同大小的内存块分配
5. 总结heap_3的适用场景

### 关键注意事项
- 由于heap_3不提供内存统计函数，需要用其他方式展示
- 需要在文档中明确说明heap_3的限制
- 强调线程安全是通过挂起所有调度器实现的

## 与heap_2项目的主要区别

| 方面 | heap_2 | heap_3 |
|------|--------|--------|
| 内存管理 | 自实现 | 标准库包装 |
| 内存统计 | 支持 | 不支持 |
| 执行时间 | 相对确定 | 不确定 |
| 内存碎片 | 有 | 取决于标准库 |
| 配置复杂度 | 低 | 需要链接器配置 |
