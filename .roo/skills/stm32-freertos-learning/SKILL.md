# STM32-FreeRTOS 渐进式学习项目 Skill

## When to use
- 在本项目 `FreeRTOSDemo` 中进行STM32-FreeRTOS学习时
- 用户请求创建新的学习示例、修改代码或编写学习文档时
- 需要按照渐进式学习思路组织内容时
- 需要重点研究堆内存管理、流缓冲区、软件定时器模块时

## When NOT to use
- 在其他非学习项目中修改FreeRTOS代码时
- 进行生产环境项目开发时
- 需要完整移植FreeRTOS到新产品时

## 本项目工作原则

1. **优先查询Demo项目原则**
    - 所有代码实现必须优先查询和参考 `FreeRTOS/FreeRTOS/Demo/` 目录中的相关项目
    - 资源配置（内存、堆栈、堆大小、时钟配置等）必须模仿Demo项目实现
    - 优先选择与当前目标平台相近的Demo项目作为参考

2. **渐进式学习**：从简单到复杂，每个学习单元只聚焦一个知识点，逐步深入
    - 每次只增加一个新概念/新API
    - 确保前一个概念理解后再进入下一个
    - 保持每个示例简洁，突出重点

3. **文档代码分离**：
    - 文档放在 `doc/` 目录下，包括学习笔记、经验沉淀
    - 示例代码放在 `projects/` 目录下，按学习阶段组织
    - 每个示例都有对应的文档说明学习要点

4. **经验沉淀**：
    - 学习过程中遇到的问题和解决方法沉淀到 `doc/experiences/` 按主题分类
    - stm32相关放到 `doc/experiences/stm32/`
    - FreeRTOS核心概念放到 `doc/experiences/freertos/`
    - 构建系统相关放到 `doc/experiences/build-system/`

5. **重点突出**：本项目重点研究三个模块：
    - **堆内存管理**：不同内存分配算法的实现和对比
    - **流缓冲区**：使用场景和实现原理
    - **软件定时器**：工作原理和使用注意事项

## 命名规范

1. **目录命名**：
   - 使用小写字母和连字符，如 `01-getting-started`
   - 保持编号两位数字，便于排序
   - 描述性名称，清晰表达目录内容

2. **文件命名**：
   - 源文件：小写字母加下划线，如 `main.c`, `freertos_config.h`
   - 文档文件：使用中文命名，如 `README.md`, "学习笔记.md"
   - 避免空格和特殊字符

3. **代码命名**：
   - 遵循FreeRTOS原生命名规范
   - 函数名使用大驼峰或FreeRTOS风格 `vFunctionName`, `xFunctionName`
   - 变量名使用小写加下划线

## 文档编写要求

1. **每个学习单元必须包含 README.md**：
   - 开头说明本章节学习的核心知识点
   - 列出学习目标
   - 说明代码结构
   - 记录遇到的问题和解决方法

2. **经验沉淀文档要求**：
   - 每个主题一个目录，放置相关文档
   - 文档开头说明解决的问题
   - 提供清晰的解决步骤
   - 保留思考过程，不只是最终答案

3. **进度文档更新**：
   - `plan/当前进度.md` 记录当前学到哪个章节
   - 下次会话自动从当前进度继续

4. **文档使用中文编写**：便于理解和交流
5. **引用来源要求**：所有教学相关内容必须注明引用来源，确保内容可信
6. **技术观点要求**：禁止无依据的推断，所有技术观点必须有官方文档或权威资料支持
7. **文档留档要求**：主要教学内容写入文档留档，不要大量输出到对话屏幕上

## 针对"不太懂虚拟机和makefile"的渐进式引导

用户明确表示不太懂虚拟机和makefile，需要按照以下方式引导：

1. **初始阶段**：
   - 使用STM32CubeIDE创建项目，不要强求用户使用makefile
   - 接受CubeIDE生成的项目结构
   - 只关注FreeRTOS内核学习，不要深入构建系统细节

2. **遇到构建问题时**：
   - 提供清晰的操作步骤，而不是要求用户自己解决makefile问题
   - 建议使用CubeIDE的图形化配置界面
   - 如果需要修改配置，指导在哪里点击修改

3. **渐进式引入**：
   - 当用户对FreeRTOS有一定理解后，再逐步引入makefile概念
   - 不要一开始就要求用户理解复杂的构建系统
   - 保持学习曲线平缓

4. **环境问题处理**：
   - 如果用户遇到环境问题，优先提供CubeIDE解决方案
   - 不要强制要求使用QEMU或虚拟机
   - 优先在硬件或CubeIDE仿真环境中学习

5. **问题排查**：
   - 引导用户一步步排查问题，先检查CubeIDE配置
   - 再检查代码错误，最后才考虑构建系统问题
   - 保持耐心，循序渐进

## FreeRTOS Demo项目查询指南

### 重要Demo项目说明

1. **CORTEX-STM32F103-Keil项目** (`FreeRTOS/FreeRTOS/Demo/CORTEX_STM32F103_Keil/`)
    - 官方STM32F103平台参考实现
    - 关键配置参数：CPU主频72MHz、系统时钟节拍1kHz、堆大小17KB、最小堆栈大小128字
    - 硬件初始化流程：HSE 8MHz → PLL ×9 → 72MHz系统时钟
    - 适合STM32F103系列的资源配置参考标准

2. **CORTEX_LM3S6965_GCC_QEMU项目** (`FreeRTOS/FreeRTOS/Demo/CORTEX_LM3S6965_GCC_QEMU/`)
    - QEMU官方支持的Cortex-M3平台
    - QEMU启动命令：`qemu-system-arm -machine lm3s6965evb`
    - 适合QEMU仿真环境的实现参考

### 查询步骤

1. 首先确定目标平台，查找最接近的Demo项目
2. 查看该项目的FreeRTOSConfig.h配置
3. 参考main.c的硬件初始化流程
4. 参考链接脚本和启动文件
5. 模仿资源配置参数（堆大小、堆栈大小等）
