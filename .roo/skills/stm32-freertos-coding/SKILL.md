# STM32 FreeRTOS 学习项目代码编写规范

## 何时使用
- 在本渐进式学习项目中编写新的FreeRTOS示例代码时
- 修改现有示例项目代码时
- 新增学习章节对应的项目时

## 何时不使用
- 不在生产环境项目中使用（本规范专为学习设计，牺牲部分工程化优化以突出学习重点）
- 不用于非学习目的的商业STM32开发

## 输入要求
- 需要明确对应教程章节编号和标题
- 需要明确本次学习的知识点和重点
- 需要遵循渐进式学习原则，逐步引入复杂度

## 文档编写要求
- 所有教学相关内容必须注明引用来源，确保内容可信
- 禁止无依据的推断，所有技术观点必须有官方文档或权威资料支持
- 主要教学内容写入文档留档，不要大量输出到对话屏幕上

## 代码编写原则

1. **优先查询Demo项目原则**
    - 所有代码实现必须优先查询和参考 `FreeRTOS/FreeRTOS/Demo/` 目录中的相关项目
    - 资源配置（内存、堆栈、堆大小、时钟配置等）必须模仿Demo项目实现
    - 优先选择与当前目标平台相近的Demo项目作为参考

2. **渐进式原则**
    - 每个项目只聚焦一个或少数几个相关知识点
    - 从简单到复杂，逐步引入新的API和概念
    - 避免一下子展示过于复杂的完整系统，让学习者专注于当前知识点

3. **注释清晰原则**
    - 每个API调用都应该解释为什么这么调用，特别是参数含义
    - 关键逻辑步骤必须有详细注释
    - 突出学习重点，标注需要注意的知识点和常见陷阱

4. **突出重点原则**
    - 将本次学习的重点代码逻辑高亮（通过注释区块分隔）
    - 非重点的基础初始化代码可以简化或引用公共部分
    - 避免无关代码干扰学习者注意力

## 项目目录结构规范

每个学习项目必须独立创建目录，结构如下：

```
projects/
└── chapter-XX-topic-name/
    ├── README.md          # 项目说明，对应教程章节，学习要点
    ├── Makefile           # 编译脚本
    ├── src/
    │   ├── main.c         # 主程序入口
    │   └── *.c/h          # 其他源文件（如果需要）
    └── doc/               # 可选：项目-specific笔记
```

- 目录命名规范：`chapter-XX-topic-name`，XX为章节号，使用小写字母和连字符
- 每个项目对应一个教程章节或一个核心知识点
- 不得多个无关知识点混在同一个项目中

## Makefile编写规范

针对STM32学习项目，Makefile需要满足：

1. **简单清晰**：避免复杂的自动化递归，让学习者能够看懂每一行
2. **明确依赖**：清晰展示编译流程，帮助学习者理解ARM程序构建过程
3. **包含常用目标**：
   - `make all` - 编译整个项目
   - `make clean` - 清理编译产物
   - `make run` - 使用QEMU运行（如果支持仿真）
   - `make debug` - 启动QEMU并等待GDB连接
4. **使用交叉编译工具**：默认使用`arm-none-eabi-`工具链
5. **注释说明**：每个关键部分都需要注释解释作用

参考模板：[assets/Makefile.template](assets/Makefile.template)

## 代码风格要求

适合学习的易读风格：

1. **命名规范**
   - 函数名使用小写字母加下划线：`void vTaskCreateExample(void)`
   - 变量名使用小写字母加下划线：`xTaskHandle xExampleTask`
   - 宏定义全大写：`#define EXAMPLE_TASK_STACK_SIZE 128`
   - 遵循FreeRTOS原生命名约定：前缀`x`表示句柄，`v`表示void返回，`pc`表示char指针等

2. **格式要求**
   - 4空格缩进，不使用Tab
   - 大括号不换行，保持紧凑便于阅读
   - 函数之间空两行，逻辑块之间空一行
   - 每行不超过80字符，方便阅读和对比学习

3. **注释要求**
   - 文件开头必须有文件说明，介绍本文件学习重点
   - 每个函数必须有函数注释，说明作用、参数、返回值
   - 关键学习点必须有详细注释解释为什么这么做
   - 常见错误用法可以用注释标注出来（`// 注意：xxx用法是错误的，因为...`）

## 与教程文档对应要求

1. **README.md必须包含**
   - 对应教程章节：链接到具体的教程文档
   - 学习目标：本项目要学习/演示哪些知识点
   - 硬件/仿真要求：是否需要特定硬件或QEMU支持
   - 运行方法：如何编译和运行
   - 预期结果：运行后应该看到什么输出

2. **代码对应**
   - 代码中的关键注释编号最好与教程讲解顺序对应
   - 新增的API调用一定要在README或注释中说明对应教程位置
   - 如果代码与教程有差异，必须在注释中说明原因

## FreeRTOS Demo项目查询指南

### 重要Demo项目说明

1. **CORTEX-STM32F103-Keil项目** (`FreeRTOS/FreeRTOS/Demo/CORTEX_STM32F103_Keil/`)
    - 官方STM32F103平台参考实现
    - 关键配置参数：
        - CPU主频：72MHz
        - 系统时钟节拍：1kHz
        - 堆大小：17KB
        - 最小堆栈大小：128字
    - 硬件初始化流程：HSE 8MHz → PLL ×9 → 72MHz系统时钟
    - 适合STM32F103系列的资源配置参考标准

2. **CORTEX_LM3S6965_GCC_QEMU项目** (`FreeRTOS/FreeRTOS/Demo/CORTEX_LM3S6965_GCC_QEMU/`)
    - QEMU官方支持的Cortex-M3平台
    - QEMU启动命令：`qemu-system-arm -machine lm3s6965evb`
    - 内存布局：Flash从0x00000000开始，SRAM从0x20000000开始
    - 适合QEMU仿真环境的实现参考

### 查询步骤

1. 首先确定目标平台，查找最接近的Demo项目
2. 查看该项目的FreeRTOSConfig.h配置
3. 参考main.c的硬件初始化流程
4. 参考链接脚本和启动文件
5. 模仿资源配置参数（堆大小、堆栈大小等）

- 编译错误：检查交叉编译工具链是否安装正确
- 链接错误：检查FreeRTOS源代码路径是否正确
