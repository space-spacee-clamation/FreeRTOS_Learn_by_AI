# 05-software-timer - FreeRTOS 软件定时器示例

## 📚 项目概述

本项目演示FreeRTOS软件定时器的使用方法，展示单次定时器和周期定时器的区别，以及定时器的启动、停止、复位操作。

**学习要点：**
- 软件定时器基础概念
- 单次定时器（One-shot Timer）
- 周期定时器（Periodic Timer）
- 定时器回调函数编写
- 定时器启动、停止、复位操作
- 定时器服务任务

## 🎯 学习目标

完成本项目后，你将能够：
1. 理解软件定时器的工作原理
2. 区分单次定时器和周期定时器
3. 编写定时器回调函数
4. 使用定时器API进行启动、停止、复位操作
5. 理解定时器服务任务的作用
6. 注意回调函数的限制（不能阻塞）


## 软件定时器的重要注意事项

### ✓ 可以在回调函数中做的：
- 轻量级计算
- 调用非阻塞的FreeRTOS API
- 发送消息到队列（非阻塞）
- 给出信号量（非阻塞）
- 操作标志组

### ✗ 不能在回调函数中做的：
- 调用阻塞函数（vTaskDelay, 阻塞的队列操作等）
- 执行耗时操作
- 访问需要互斥保护的共享资源（没有合适的保护）
- 调用可能导致上下文切换的函数

### 为什么有这些限制？
- 回调函数在定时器服务任务中执行
- 阻塞会影响其他定时器的执行
- 定时器服务任务的优先级通常较高，不应该被长时间占用

## 软件定时器的适用场景

软件定时器适合以下场景：

✓ **周期性任务** - 需要定期执行的操作
✓ **延时操作** - 非精确的延时需求
✓ **超时检测** - 监控操作是否超时
✓ **状态轮询** - 定期检查状态变化
✓ **简单调度** - 多任务的简单时间调度

软件定时器不适合：

✗ **高精度定时** - 精度依赖系统滴答，不适合高精度要求
✗ **阻塞操作** - 回调函数不能阻塞
✗ **复杂处理** - 回调中执行复杂处理会影响系统
✗ **硬件相关** - 需要硬件定时器的场合

## 单次定时器 vs 周期定时器对比

| 特性 | 单次定时器 | 周期定时器 |
|------|-----------|-----------|
| 触发次数 | 一次 | 无限次（直到停止） |
| 创建参数 | pdFALSE | pdTRUE |
| 自动重载 | ✗ 否 | ✓ 是 |
| 适用场景 | 延时操作、超时 | 周期性任务、轮询 |
